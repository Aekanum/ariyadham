# Internationalization (i18n) - Ariyadham

**Story 8.1: i18n Setup**
**Status:** Foundation Complete
**Date:** 2025-11-09

## Overview

Ariyadham supports Thai and English languages. The i18n infrastructure is built using a custom context-based solution optimized for Next.js App Router.

## Quick Start

### Using translations in components

```typescript
'use client';
import { useTranslation } from '@/contexts/LanguageContext';

export default function MyComponent() {
  const { t } = useTranslation('common');

  return (
    <div>
      <h1>{t('app.name')}</h1>
      <p>{t('app.tagline')}</p>
    </div>
  );
}
```

### Language Switcher

```typescript
import LanguageSwitcher from '@/components/layout/LanguageSwitcher';

<LanguageSwitcher />
```

## Architecture

- **Locales:** `th` (Thai - default), `en` (English)
- **Translation Files:** `/public/locales/{locale}/{namespace}.json`
- **Namespaces:** common, auth, reader, author, admin
- **Context:** `LanguageProvider` in root layout
- **Hooks:** `useLanguage()`, `useTranslation(namespace)`

## Translation Files

Located in `/public/locales/`:
- `th/common.json` - Thai common UI
- `en/common.json` - English common UI
- `th/auth.json` - Thai authentication
- `en/auth.json` - English authentication
- `th/reader.json` - Thai reader features
- `en/reader.json` - English reader features

## Adding Translations

1. Add key-value pairs to the appropriate JSON file
2. Use dot notation for nested values: `app.name`, `nav.home`
3. Use placeholders for dynamic values: `{count}`, `{name}`

Example:
```json
{
  "welcome": "Welcome {name}",
  "articleCount": "{count} articles"
}
```

Usage:
```typescript
t('welcome', { name: 'User' }) // "Welcome User"
t('articleCount', { count: 5 }) // "5 articles"
```
